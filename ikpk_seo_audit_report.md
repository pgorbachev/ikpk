# Технический и SEO-аудит IKPK — Полный отчёт (black-box)

> **Дата:** 31 марта 2026  
> **Режим:** Black-box (без доступов к аналитике, CMS, серверу)  
> **Основной объект:** https://ikpk.su  
> **Связанные домены:** medshop.ikpk.su, kinezio.shop, ikpk812.ru, mudriydoctor.ru

---

## Executive Summary

**Вердикт: сайт можно и нужно ремонтировать. Полный rebuild НЕ требуется.**

Основной сайт ikpk.su построен на Next.js с серверным рендерингом — это хорошая технологическая база. Все 254 страницы отдают 200, canonical на месте, title/description/H1 заполнены на 100% страниц. SEO-оценка Lighthouse стабильно 92/100.

Однако есть **3 критических проблемы**, которые напрямую бьют по трафику:

1. 🔴 **Производительность на мобильных катастрофическая** — Performance score 36-56/100, LCP до 9.8 сек, TBT до 2070ms. Google учитывает mobile-first, и это прямой фактор ранжирования.

2. 🔴 **Нет Sitemap в robots.txt** — поисковики не находят sitemap автоматически. При 254 страницах это критично.

3. 🔴 **16 страниц-сирот** (преподаватели и видео-плейлисты) не имеют входящих внутренних ссылок. Не критично для трафика, но стоит исправить.

**Дополнительные риски:**
- Ссылка на staging.ikpk.su в HTML главной страницы
- Два интернет-магазина (medshop.ikpk.su и kinezio.shop) каннибализируют друг друга
- kinezio.shop — в title утечка «на woocommerce»
- Нет EducationalOrganization/Course разметки (упущенные rich snippets)
- 6 из 7 security headers отсутствуют

**Оценка по стратегии:** 90-дневный план исправлений позволит решить все критические проблемы. Rebuild можно отложить на 12-18 месяцев и вернуться к вопросу только если восстановление трафика не произойдёт.

| Проблема | Severity | Пояснение |
|----------|----------|-----------|
| **Нет директивы Sitemap** | 🔴 Critical | `Sitemap: https://ikpk.su/sitemap.xml` отсутствует. Поисковики не узнают о sitemap автоматически |

| `Disallow: /*.pdf` | 🟡 Medium | PDF-документы (лицензии, программы) не индексируются. Если есть полезные PDF — это потеря трафика |

## 2. Технические проблемы

### 2.1 Редиректы

| Тест | Результат | Оценка |
|------|-----------|--------|
| http://ikpk.su | 308 → https://ikpk.su | ✅ OK |
| http://www.ikpk.su | 308 → https://www.ikpk.su → 308 → https://ikpk.su | 🟡 2 хопа (цепочка) |
| https://www.ikpk.su | 308 → https://ikpk.su | ✅ OK |
| Trailing slash | 308 → без слеша | ✅ OK |

**Проблема:** `http://www.ikpk.su` делает 2 редиректа вместо 1. Нужно перенаправлять сразу на `https://ikpk.su`. Severity: 🟢 Low (маловероятный паттерн для пользователей).

✅ Все возвращают корректный 404. **Но тело ответа ~100 KB** — это слишком много для 404-страницы (полный рендер SPA).

| Проблема | Severity |

## 8. Мультидоменность и каннибализация

### 8.1 Перекрёстные ссылки

```
ikpk.su → kinezio.shop (в навигации, на каждой странице)
ikpk.su → mudriydoctor.ru (в навигации, на каждой странице)

```

### 8.2 Каннибализация medshop.ikpk.su vs kinezio.shop

🔴 **Два магазина продают одни и те же книги:**
- medshop.ikpk.su — WordPress/WooCommerce (Woodmart)
- kinezio.shop — WordPress/WooCommerce

Оба домена конкурируют друг с другом в выдаче. При этом kinezio.shop — единственный, на который ведут ссылки из навигации ikpk.su.

| Проблема | Severity |
|----------|----------|
| Два магазина с дублирующим ассортиментом | 🔴 High |
| medshop.ikpk.su не связан с навигацией ikpk.su | 🟡 Medium |
| Утечка SEO-веса через ссылку на Яндекс.Диск (Фото) | 🟡 Medium |

### 8.3 Рекомендации

1. **Закрыть один из магазинов**, сделать 301-редирект на другой
2. Если нужны оба — настроить canonical между ними
3. Убрать ссылку на Яндекс.Диск из навигации или добавить `rel="nofollow"`

### 9.3 Ключевые разрывы (где ikpk.su отстаёт)

1. **Отсутствие OG-тегов** — конкуренты имеют полную разметку для соцсетей
2. **Нет SEO-плагина** — у конкурентов Yoast/Rank Math автоматически генерирует sitemap, OG, schema, canonical
3. **Шаблонные description** vs уникальные у конкурентов
4. **Нет Course/Event schema** — конкуренты через плагины получают rich snippets
5. **Custom CMS** означает, что каждое SEO-улучшение требует разработки, а у конкурентов — настройки плагина

### 9.4 Где ikpk.su сильнее

1. **Уникальный контент** — эксклюзивный представитель институтов Апледжера и Барраля в России
2. **Современная технология** — Next.js быстрее WordPress при правильной настройке
3. **254 страницы** — обширный каталог курсов
4. **Чистые URL** — без мусорных параметров

## Рекомендация

### ✅ Ремонтируем текущий сайт

**Обоснование:**

1. **Технологическая база хорошая.** Next.js с SSR — это лучше, чем WordPress, при правильной настройке. Сайт рендерит HTML на сервере, все страницы доступны ботам, canonical на месте.

2. **Контент на месте.** 254 страницы с уникальным title, description и H1. Нет дублей (кроме 1 пары). Нет thin content.

3. **Проблемы решаемые.** Все критические проблемы (сиротские страницы, sitemap в robots.txt, производительность, schema) — это инженерные задачи, а не архитектурные.

4. **Rebuild дороже и рискованнее.** Миграция 254 URL с сохранением SEO-позиций — сложный процесс. При ошибке трафик просядет ещё сильнее.

5. **Ожидаемый срок восстановления:** 2-4 месяца после внедрения исправлений из плана 0-30 дней.

**Когда пересмотреть решение:** если через 4 месяца после внедрения всех исправлений трафик не вырос на 20%+, стоит рассмотреть частичный rebuild (переход на WordPress + Yoast для упрощения SEO-управления).
