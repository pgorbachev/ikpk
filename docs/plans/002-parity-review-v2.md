## Проблема

Ревью выявило 3 ключевых разрыва между новым и оригинальным сайтом:

1. **[P1] Главная урезана** — потеряны блоки «Наши преимущества» (6 карточек), «Наш подход к обучению» (описание + статистика 3 показателя), «Наши программы» (3 института с описаниями). Текущая главная: Hero → News → Institutes → Promotions → Newsletter.

2. **[P1] Legacy HTML без CSS** — на контентных страницах body_html содержит ~30 уникальных CSS-module префиксов (typography_*, articles-form_*, collapsible_*, schedule-prices_* и т.д.) без соответствующих стилей. Результат: «плоский» контент без структуры.

3. **[P2] Две визуальные системы** — главная в новом дизайне, контентные страницы — сырой legacy HTML. Визуально выглядит как два разных сайта.

## Стратегия

**Компонентный rebuild** (не legacy-CSS-прослойка), потому что:
- Legacy классы — CSS Modules с хешами, нет смысла воспроизводить
- Структурированные данные уже в entities JSON, можно рендерить компонентами
- Поддерживаемость: свои компоненты с Astro scoped styles >> raw HTML с мёртвыми классами

## Задачи (скорректированный порядок)

### 1. static-page-cleanup — Очистка body_html с unit-тестами [P1] ← ПЕРВЫЙ

**Scope:** создать `cleanBodyHtml()` с **явными, тестируемыми правилами трансформации**.

**Правила трансформации:**

| Что | → Что | Пример |
|-----|-------|--------|
| Целые блоки с `class*=subscribe-news-form_\|PhoneInput`  | → удалить целиком (UI-формы подписки/телефона) | `<div class="subscribe-news-form_wrapper___xXSq">...</div>` → ∅ |
| `<button class*=button_>`, `<input class*=text-field_>`, `<label class*=checkbox_>` **только внутри** контейнеров форм (subscribe-news-form, seminar-form, contacts-form) | → удалить (UI-элементы форм в контексте) | |
| Элементы с `class*=articles-form_articleContainer\|seminar-form_\|teachers-form_\|cooperation_` | → unwrap (оставить children) | `<div class="articles-form_articleContainer__13oDC"><p>text</p></div>` → `<p>text</p>` |
| `<div class="collapsible_*">` с вложенным заголовком + контентом | → `<details><summary>{заголовок}</summary>{контент}</details>` | |
| `<h1-h6 class="typography_*">` | → тот же тег без class | `<h2 class="typography_h2__Dgg2h">Title</h2>` → `<h2>Title</h2>` |
| `<p/span class="typography_*">` | → тот же тег без class | |
| `<div class="se-root">` | → unwrap (оставить содержимое) | |
| Все остальные CSS-module классы (содержат `__`) | → strip class attribute | `<div class="foo__bar">` → `<div>` |

**Unit-тесты:** 5–6 HTML-фикстур (article, seminar, kontakty, oplata, svedeniya) — вход/выход cleanBodyHtml().

Файлы: `web/src/lib/html-cleaner.ts`, `web/tests/html-cleaner.test.ts`
