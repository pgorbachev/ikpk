## TD-1 — Дублирование логики пагинации в statyi/index.astro

**Файлы:** `web/src/pages/statyi/index.astro`
**Метки в коде:** `TODO(TD-1)` (2 места — серверная `buildPaginationTokens` и клиентская `getPaginationTokens`)

**Проблема:**
Алгоритм построения токенов пагинации продублирован дважды: серверная функция для SSG initial render и клиентская для динамической перестройки при фильтрации. При изменении алгоритма нужно обновлять оба места.

**Решение:**
Вынести в `web/src/lib/pagination.ts` как чистую функцию и импортировать из обоих мест (Astro компилирует `<script>` с поддержкой ESM).

## TD-2 — Все статьи как `<template>` в HTML на странице /statyi

**Файлы:** `web/src/pages/statyi/index.astro`
**Метки в коде:** `TODO(TD-2)`

**Проблема:**
Клиентская фильтрация/поиск работает через скрытые `<template>` теги — все статьи рендерятся в HTML при сборке. При росте каталога страница будет тяжёлой (сейчас 68 статей — приемлемо).

**Решение (на выбор):**
1. Статические пагинированные страницы `/statyi/page/2` — нет JS-поиска.
2. JSON-индекс `/statyi-index.json` — клиент грузит один раз, рендерит карточки динамически. Сохраняет поиск/фильтрацию без раздутого HTML.

**Когда актуально:** при росте каталога свыше ~200 статей.
