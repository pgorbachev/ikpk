## TD-2 — Все статьи как `<template>` в HTML на странице /statyi

**Файлы:** `web/src/pages/statyi/index.astro`
**Метки в коде:** `TODO(TD-2)`

**Проблема:**
Клиентская фильтрация/поиск работает через скрытые `<template>` теги — все статьи рендерятся в HTML при сборке. При росте каталога страница будет тяжёлой (сейчас 68 статей — приемлемо).

**Решение (на выбор):**
1. Статические пагинированные страницы `/statyi/page/2` — нет JS-поиска.
2. JSON-индекс `/statyi-index.json` — клиент грузит один раз, рендерит карточки динамически. Сохраняет поиск/фильтрацию без раздутого HTML.

**Когда актуально:** при росте каталога свыше ~200 статей.

## TD-3 — Пагинация /statyi без fallback при отключённом JS

**Файлы:** `web/src/pages/statyi/index.astro`
**Метки в коде:** нет (архитектурный долг)

**Проблема:**
Пагинация реализована через `<button>` + клиентский JS. При отключённом JavaScript или ошибке загрузки скрипта пользователь и поисковый бот видят только первые 6 статей — нет ни ссылок, ни URL для остальных страниц.

**Решение (на выбор):**
1. Статические страницы `/statyi/page/2`, `/statyi/page/3` — работают без JS, индексируются поисковиками.
2. `<a href="?page=2">` с прогрессивным улучшением через JS — сохраняет URL-навигацию и работает без скриптов.

**Когда актуально:** сейчас некритично (SEO покрывается SSG), но желательно до роста каталога.
